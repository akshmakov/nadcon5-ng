19 ===========
20

25 Control File {#scontrolfile}
26 ------------
27 
28 
29 The Control File describes the [In Files](#sinfiles) which contain data for a given transformation.
30 
31 They are named
32 
33  control.OLD_DATUM.NEW_DATUM.REGION
34 
35 Control Files exist in the folder `data/Control` . The existence of a control file means that the transformation is defined.
36 
37 \note Conversely. If a control file does not exist, then the transformation is not defined
38 
39 
40 The Structure of the Control File is a list of valid input files with formatted header information.
41 
42 We show this structure by using an example.
43 
44  HEADER: Master File for creating a NADCON5 work file
45  REGION: CONUS
46  DATUM1: USSD
47  DATUM2: NAD27
48  REJMET: 10000
49  NFILES: 49
50  NADCON5.USSD.NAD27.AL.in
51  NADCON5.USSD.NAD27.AR.in
52  ...
53  49 Total Files Listed
54  ...
55 
56 
57 The Control file is used by \ref makework to generate the initial [Work File](#sworkfile)
58 
59 \note It is critical that the header labels (`HEADER`, `REGION`, `DATUM1`, etc.)
60 are not changed, as the program \ref makework verifies each line against these strings.
61 
62 
63 **Parameter Details**
64 
65  - `HEADER` - Header line. Can contain anything.
66  - `REGION` - REGION Parameter from [Build Pipeline](#sdatapipeline).
67  Must conform to the following list:
68  - conus
69  - alaska
70  - prvi
71  - hawaii
72  - guamcnmi
73  - as
74  - pribilof
75  - stlawrence
76  - `DATUM1` - The older datum, chronologically.
77  `OLD_DATUM` Parameter from [Build Pipeline](#sdatapipeline).
78  - `DATUM2` - The newer datum, chronologically.
79  `NEW_DATUM` Parameter from [Build Pipeline](#sdatapipeline).
80  - `REJMET` The rejection criteria in meters.
81  Basically if any latitude shift or longitude
82  shift or horizontal shift exceeds this value
83  (in absolute value), then all shifts for this
84  point are set to zero (to avoid asterisks in
85  the output file) but the whole line is labeled
86  with a triple reject criteria, effectively
87  eliminating the pair from use.
88  - `NFILES` - The number of *.in files which connect
89  the old and new datums in the region being
90  addressed.

93 workedits File {#sworkedits}
94 --------------
95 
96 Used by \ref makework and located in \ref data/Work , the `workedits` is a database of rejected datum transformation points. That is, the `workedits` file provides a list of specific points in the [InFiles](#sinfiles) which should be rejected for various, manually identified, reasons.
97 
98 The file format is fixed column delminted. The formatting is hardcoded in \ref makework with index records, the format each individual read is a complete string. Strings are not "stripped" so left justification is required.
99 
100 The columns are:
101 
102  01- 10 : olddtm : lower case, left justified
103  11 : "|" : vertcal spacer just for ease of reading
104  12- 21 : newdtm : lower case, left justified
105  22 : "|" : vertcal spacer just for ease of reading
106  23- 32 : region : lower case, left justified (conus, alaska, hawaii, prvi, guamcnmi, as)
107  33 : "|" : vertcal spacer just for ease of reading
108  34- 39 : PID : upper case, left justified
109  40 : "|" : vertcal spacer just for ease of reading
110  41- 43 : rejects: Three digits (0's or 1's only) to reject lat, lon, eht, in that order. '1' = reject, '0' = keep
111  44 : "|" : vertcal spacer just for ease of reading
112  45-200 : reason : Upper/lower case, giving first your name then reason for the line to exist
113 
114 
115 
116 Or, in other Words:
117 
118  olddtm |newdtm |region |PID |rej|Reason
119  lwr case |lwr case |lwr case |uprcas|0's|Give your name or initials first, then reason
120  left |left |left |6 char|or |Upper or lower case or both
121  justified|justified |justified | |1's|
122  10 char |10 char |10 char | | |
123 
124 
125 An Example:
126 
127  ussd |nad27 |conus |KG0640|110|DAS: USSD and NAD 27 coordinates are impossibly identical.
128  ussd |nad27 |conus |DK3691|110|DAS: USSD and NAD 27 coordinates are impossibly identical.
129  XXXXXXXXXX-XXXXXXXXXX-XXXXXXXXXX-XXXXXX-XXX-XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX
130  oldtm newtm region pid rej comment
131 
132 
133 
134 
135 
136 
137 
138 
139 
140 
141 
142 
143 
144 **Parameters:**
145 
146  - `oldtm` - Source Datum
147  - `newtm` - Target Datum
148  - `region` - Geographical Region (e.g. conus)
149  - `pid` - Point ID
150  - `rej` - Bit Field to reject lat, lon, eht
151  - `Reason` - Comment Field

154 In Files {#sinfiles}
155 -------
156 
157 The In Files are located in \ref data/InFiles and contain the actual reference and transformation data for the construction of the NADCON5.0 dataset.
158 
159 These files are created per state/territory and for every transformation vector.
160 
161 The files are named per the convention
162 
163  NADCON5.DATUM1.DATUM2.STATE.in
164 
165 
166 
167 
168 The format of these files is a fixed format file. The format strings are defined in \ref makework
169 
170 The first row of the file identifies the converted datums in comment header, with a Fortran Format specifier of
171 
172  100 format(27x,a15,26x,a15)
173 
174 For example: (with format offsets and variable name indicated underneath)
175 
176  US Standard Datum | NAD 27
177  ---------------------------XXXXXXXXXXXXXXX--------------------------XXXXXXXXXXXXXXX
178  nameh namef
179 
180 
181 That is, 15 Character Strings with offsets `27` and `68` (`27+15+26`).
182 
183 \note Characters outside the defined 15 Character Region are simply ignored, in the above example, the String `US Standard Datum` would be truncated and trimed by Fortran to `Standard Datum`. However, this header line is not currently used for anything by \ref makework
184 
185 The remaining rows define the actual datapoints, with a fortran format specifier of
186 
187  101 format(a6,1x,a2,5x,a13,1x,a14,1x,a9,3x,a13,1x,a14,1x,a9)
188 
189 For Example:
190 
191  AA5496 CA 085 N372639.93300 W1220955.92000 N/A | N372639.71836 W1220959.79579 N/A
192  XXXXXX-XX-----XXXXXXXXXXXXX-XXXXXXXXXXXXXX-XXXXXXXXX---XXXXXXXXXXXXX-XXXXXXXXXXXXXX-XXXXXXXXX
193  pid state clath clonh cehth clatf clonf cehtf
194 
195 
196 
197 
198 **Parameters:**
199 
200  - `pid` - Point ID (NGS Internal Unique Designator)
201  - `state` - State this point belongs to
202  - `clath` - Source Datum Lat. (Decimal Degrees with Cardinal Direction)
203  - `clonh` - Source Datum Lon. (Decimal Degrees with Cardinal Direction)
204  - `cehth` - Source Datum Height (Meters)
205  - `clatf` - Target Datum Lat.
206  - `clonf` - Target Datum Lon.
207  - `cehtf` - Target Datum Height
208 
209 
210 \note Partial Transformation points (e.g. no Height data) are specified by using `N/A` specifier to exclude the lat. , lon. , and/or height from the calculations.
211 
212 The file is read until the last record indicated by an end of file (`EOF`), there is no footer or record counter.

229 Work File {#sworkfile}
230 --------
231 
232 The `workfile` is the first file created in the build pipeline. It serves as a local database of all the points and associated transformation data which is later used to construct images, grids, and other outputs.
233 
234 The workfile is generated with the name
235 
236  work.DATUM1.DATUM2.REGION
237 
238 And is placed (hardcoded) in the local `Work/` directory
239 
240 
241 
242 The new file has the following format:
243 
244 
245  Cols Format Description
246  1- 6 a6 PID
247  7 1x - blank -
248  8- 9 a2 State
249  10 a1 Reject code for missing latitude pair (blank for good)
250  11 a1 Reject code for missing longitude pair (blank for good)
251  12 a1 Reject code for missing ellip ht pair (blank for good)
252  13 1x - blank -
253  14- 27 f14.10 Latitude (HARN), decimal degrees (-90 to +90)
254  28 1x - blank -
255  29- 42 f14.10 Lonitude (HARN), decimal degrees (0 to 360)
256  43 1x - blank -
257  44- 51 f8.3 Ellipsoid Height (HARN), meters
258  52 1x - blank -
259  53- 61 f9.5 Delta Lat (FBN-HARN), arcseconds
260  62 1x - blank -
261  63- 71 f9.5 Delta Lon (FBN-HARN), arcseconds
262  72 1x - blank -
263  73- 81 f9.3 Delta Ell Ht (FBN-HARN), meters
264  82 1x - blank -
265  83- 91 f9.5 Delta Horizontal (absolute value), arcseconds
266  92 1x - blank -
267  93-101 f9.5 Azimuth of Delta Horizontal (0-360), degrees
268  102 1x - blank -
269  103-111 f9.3 Delta Lat (FBN-HARN), meters
270  112 1x - blank -
271  113-121 f9.3 Delta Lon (FBN-HARN), meters
272  122 1x - blank -
273  123-131 f9.3 Delta Horizontal (absolute value), meters
274 
275 
276 Which is defined by fortran `format` Identifier
277 
278  104 format(a6,1x,a2,a1,a1,a1,1x,f14.10,1x,f14.10,1x,f8.3,1x,
279  * f9.5,1x,f9.5,1x,f9.3,1x,f9.5,1x,f9.5,1x,f9.3,1x,f9.3,1x,f9.3,
280  * 1x,a10,1x,a10)
281
