10 ## Makefile Primer
11 
12 **Note:** This primer has been adapted from other sources and is not specific to `NADCON5-ng`
13 
14 Makefiles define **rules** to make **targets**.
15 
16 A Makefile may look something like this.
17 
18 
19 
20  # Target to Compile a single file.
21  # The first line defines the target
22  # further lines define the commands
23  # that are run
24  object.o:
25  cc -o object.o object.c
26 
27  # Target to link executable with external releaselib.
28  # object.o is a dependency of release_exec
29  release_exec: object.o
30  ld -r -o release_exec releaselib object.o
31 
32  # Convenience Target
33  release: release_exec
34 
35  # Since "release" does not actually produce a file
36  # this is required boiler plate
37  .PHONY: release
38 
39 
40 This Makefile is actually equivalent to a simple compiler one-liner.
41 
42 
43  cc -o release_exec -lreleaselib object.c
44 
45 
46 However, it already provides power behind the scenes.
47 
48 For example Calling
49 
50 
51  make release
52 
53 
54 Will not recompile the executable if `object.c` has not been changed.
55 
56 The real power of `Make` comes from "Pattern Rules" constructed using [Automatic variables](https://www.gnu.org/software/make/manual/html_node/Automatic-Variables.html)
57 and [Pattern Matching](https://www.gnu.org/software/make/manual/html_node/Pattern-Match.html#Pattern-Match) which result in a more general rule.
58 
59 
60  # The target name is special
61  # and the variable $@
62  # refers to the target name
63  test.o:
64  cc -o $@ object.c
65 
66  # The dependency name is special
67  # and the variable $< refers to the
68  # first (only) dependency
69  better_test.o: object.c
70  cc -o $@ $<
71 
72  # Multiple Dependencies
73  # can be refered to by the variable
74  # $^
75  other_test.a: test.o better_test.o
76  ld -r -o $@ $^
77 
78  # Pattern matching makes general rules
79  # The following rule compiles all .c files to .o files
80  %.o:%.c
81  cc -o $@ $<
82 
83  # PHONY rules can be used to build multiple
84  # dependencies
85  build_some_libs: pattern_lib.o \
86  pattern_lib2.o \
87  pattern_lib3.o \
88  other_test.a
89  .PHONY: build_all_libs
90

92 
93 By default `Make` assumes that the target is a real file that is created, and will track the state of
94 dependency based on the file and if it exists.
95 
96 As such when dealing with build directories, a target must be specified with its path for make to track its status.
97 
98 The `.PHONY` target is a special target that indicates that
99 it should not be tracked and will be rebuilt every time, often this is good for things like printing debug output
100 or a convenience target that builds multiple targets (e.g. `all`).
101 
102 
103 In our Makefiles you will see things like the following, this example is our default rule for building fortran programs
104 
105  $(BIN_DIR)/%:$(notdir %).f | $(BIN_DIR)
106  $(FC) $(FFLAGS) $< -o $@
107 
108 
109 
110 In this case, `FC`, `FFLAGS`, are variables define in our Makefile which can be overridden at the command line, this is described below. `BIN_DIR` is a Makefile Variable and is the directory where binary programs artifacts are to be placed after building.
111 
112 This default rule maps all programs like `/path/to/build/bin/myprogram` to compile from the file `myprogram.f` using the Fortran Compile `FC` and the compilation flags `FFLAGS`
